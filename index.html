Windows系统下安装NativeScript

2016-11-21
NativeScript

近来打算使用Angular2 & NativeScript折腾点东西，原以为能很快完成开发环境的搭建，却不曾想在Windows下安装NativeScript CLI时便遇到了极大的挑战。

目标

目标是在Windows8上安装Angular2 & NativeScript的Android开发环境。以下是官方的系统需求：

  • Windows 7 Service Pack 1 or later
  • The latest Node.js 4.x, 6.x or 7.x stable official release
  • JDK 8 or a later stable official release
  • Android SDK 22 or a later stable official release
  • Android SDK Build-tools 23.0.3 or a later stable official release

设置环境变量JAVA_HOME & ANDROID_HOME。

安装步骤

满足上面的系统需求后，就可以安装NativeScript CLI了，只需运行npm命令即可。

npm安装包虽然很简便，但是由于NativeScript需要编译Node模块，而我的Windows系统没有相关的编译工具，自然无法顺利完成安装。

出现的问题

这是因为依赖的node-gyp是一个跨平台的用来编译Node.js上native addon的命令行工具，而这个工具需要所在的Windows上具有相关的编译工具。

问题发生后，简单在网上一搜，网友们给出的方案大都是安装VS！大家都知道微软的VS的硬盘空间需求量是多么的大，我不想为了一部分编译工具而安装那么庞大的软件。后来仔细定位问题，最终在node-gpy的github主页上找到了解决方案。官方提供的解决方式有：

  1. 在CMD(管理员权限运行)中使用命令npm install --global --production windows-build-tools自动安装，这会自动安装python以及VC++ Build Tools
  2. 使用默认的安装选项安装Visual C++ Build Tools
  3. 安装Visual Studio 2015

我选择手动安装Visual C++ Build Tools之后，NativeScript得以成功安装。

在命令终端输入指令验证一下：tns doctor。

结束语

在NativeScript安装完成之后，迫不及待的通过其官方文档，生成了一个ns & ng2工程，只需要几个NativeScript CLI命令即可完成。

  • tns create myApp --ng
  • tns platform add android
  • tns run android

-END-
